# Atlas Packing / Texture Atlas Export v17

v17 adds support for Suwol2D atlas metadata generated by the Electron editor.

## Export Shape

The editor can optionally create:

```text
Atlas/
  character.atlas.png
  character.atlas.json
```

The `.suwol2d` file keeps attachment `image` fields unchanged and adds an
optional top-level `atlases` array. Existing files without `atlases` still load
through the individual `Textures/` fallback.

## Importer Behavior

The importer searches for atlas textures in nearby `Atlas/` folders and adds
them to the generated prefab texture list. It also keeps individual texture
lookup behavior for compatibility.

## Runtime Behavior

At render time the runtime checks whether an attachment image name matches an
atlas region. When it does, the renderer uses the atlas texture and remaps UVs:

```text
finalU = region.u + vertex.u * (region.u2 - region.u)
finalV = region.v + vertex.v * (region.v2 - region.v)
```

If no region matches, the renderer uses the previous individual texture lookup.

## Limits

- No rotated atlas regions
- No trimmed atlas regions
- No multi-page atlas
- PNG atlas export only in the Electron editor
- No Spine compatibility
